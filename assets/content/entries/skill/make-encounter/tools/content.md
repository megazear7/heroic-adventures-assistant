# Make Encounter

## Purpose

Build encounters that are tactically interesting, level-appropriate, and easy to adjust live at the table.

## When to Use

- Session prep for combat or mixed challenge scenes.
- Need a fast replacement encounter mid-session.
- GM requests encounter variants for different difficulty targets.

## Required Inputs

- Party level, size, and class mix.
- Desired difficulty tier and encounter purpose.
- Environment/terrain context.
- Story constraints (must survive NPC, capture objective, timer pressure, etc.).

## Workflow

1. Define objective beyond defeat-all when possible (hold, escape, disable, rescue, delay).
2. Set threat assumptions using party action economy and durability profile.
3. Choose monster role composition with clear battlefield jobs.
4. Add terrain features, movement constraints, and line-of-sight considerations.
5. Add pacing events: reinforcements, hazards, clock triggers, morale shifts.
6. Prepare two tuning levers: one for downshift, one for upshift.
7. Add quick adjudication notes for likely edge cases.

## Quality Checks

- Encounter has a clear win/fail state.
- Enemy composition creates pressure without hard-locking agency.
- Terrain meaningfully affects decisions.
- Difficulty can be tuned in under one round.

## Output Template

Return:
- Encounter summary (goal, location, stakes).
- Enemy roster by role and function.
- Terrain/objective map notes.
- Trigger timeline (round/condition-based).
- Downshift/upshift knobs.

## Missing Information Policy

If party data is partial, use moderate baseline assumptions, label them explicitly, and include a quick calibration note the GM can apply after round 1.

## Rule Routing

- Ground design in encounter difficulty, monster roles, initiative, movement/positioning, and attacks/defenses rule skills.
- If balance is uncertain, default to safer pressure and provide escalation toggles.
