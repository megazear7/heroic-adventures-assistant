# Generate Adventure

## Purpose

Create playable adventures with clear stakes, structured scenes, and pacing that respects party capability and resource pressure.

## When to Use

- GM requests a one-shot outline.
- Campaign needs a new arc segment.
- Session prep requires encounters, objectives, and rewards in one package.

## Required Inputs

- Party level, size, and composition.
- Adventure length target (single session, short arc, multi-session).
- Tone/theme and setting constraints.
- Desired challenge intensity and narrative focus.

## Workflow

1. Define premise: hook, antagonist force, immediate stakes, and failure cost.
2. Establish objective path with 3-5 scenes (or phase map for arcs).
3. Assign scene function for each beat: discovery, social pressure, traversal, conflict, climax, fallout.
4. Add encounter plan with escalation triggers and fallback de-escalation knobs.
5. Add non-combat gates (clues, negotiations, hazards, logistics) with clear success/failure effects.
6. Build reward cadence: short-term loot, narrative rewards, and progression-safe outcomes.
7. End with continuity hooks for next session/arc.

## Quality Checks

- Stakes are explicit and time-pressured.
- At least one alternate path exists when players bypass expected scenes.
- Encounter pressure matches party level and action economy reality.
- Rewards do not create abrupt power spikes.

## Output Template

Return:
- Adventure summary (hook, stakes, antagonist pressure).
- Scene-by-scene plan with objectives and likely player choices.
- Encounter blocks with escalation/reinforcement notes.
- Reward and consequence table.
- GM pacing knobs (accelerate, hold, escalate).

## Missing Information Policy

If campaign details are missing, use a neutral fantasy baseline, note assumptions, and provide quick reskin tags for names, factions, and locations.

## Rule Routing

- Use encounter-building, monster-role, action-economy, and loot/economy rule skills for mechanical grounding.
- If a required rule reference is absent, provide a conservative suggested ruling and mark it clearly.
