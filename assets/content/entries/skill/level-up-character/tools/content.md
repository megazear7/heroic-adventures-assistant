# Level Up Character

## Purpose

Advance characters legally and efficiently while preserving build identity and improving table performance.

## When to Use

- A character gains a level.
- A player wants advancement recommendations before confirming picks.
- A GM needs legality checks on submitted level-up changes.

## Required Inputs

- Current character sheet and level.
- Existing feature history and prior choices.
- Player goals (damage, survivability, utility, support, theme).
- Campaign constraints and expected threats.

## Workflow

1. Verify baseline: current level, prior class/race/background selections, and locked choices.
2. Apply mandatory level gains in official order.
3. Present legal options at this level only.
4. Evaluate options against role goals and party coverage.
5. Detect and flag prerequisite failures, redundant picks, or anti-synergy.
6. Finalize chosen features and recalculate derived profile notes.
7. Provide short tactical changes for the new level.

## Quality Checks

- No feature is taken before its level gate.
- Prerequisites and dependencies are satisfied.
- Updated actions include correct action-type labeling where relevant.
- Build remains internally coherent with campaign role.

## Output Template

Return:
- Level-up change log (before -> after).
- New legal selections and why chosen.
- Updated combat/utility role summary.
- New action options with action-type tags where applicable.
- Next-level planning notes (1-2 recommended paths).

## Missing Information Policy

If prior sheet details are incomplete, halt irreversible choices, provide a provisional plan, and list exact missing fields needed to finalize legally.

## Rule Routing

- Prioritize leveling/progression, class identity, race features, equipment dependencies, and action economy rule skills.
- Never grant level-gated benefits without confirming legal eligibility.
