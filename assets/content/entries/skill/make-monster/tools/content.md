# Make Monster

## Purpose

Create monsters that are readable, role-faithful, and balanced for intended party level and encounter function.

## When to Use

- New creature is needed for a custom setting or arc.
- Existing creature needs a variant or reskin.
- Encounter design requires a missing role in a monster lineup.

## Required Inputs

- Target level band and encounter role.
- Desired fantasy/theme and behavior pattern.
- Encounter context (solo, paired, minion support, boss phase).
- Complexity target (simple, standard, elite-style complexity).

## Workflow

1. Define creature concept and role promise (what pressure it creates).
2. Build profile aligned to role expectations at target level.
3. Add signature actions with explicit action-type tags.
4. Add one reactive or positional behavior if role supports it.
5. Ensure counterplay exists (telegraph, weakness, tradeoff, cooldown-like pacing).
6. Add usage notes: ideal allies, terrain fit, and encounter deployment tips.

## Quality Checks

- Damage/defense pressure matches level intent.
- Action economy is complete but not overloaded.
- Abilities are distinct and non-redundant.
- Counterplay is visible and exploitable.

## Output Template

Return a monster card with:
- Name, role, level band, brief fantasy tag.
- Profile summary.
- Action list with action-type labels.
- Passive/reactive traits.
- Counterplay notes.
- GM deployment notes.

## Missing Information Policy

If role or level is unclear, offer two safe variants (lower-pressure and standard-pressure) and mark which one is default.

## Rule Routing

- Ground profile and actions in monster-role, attacks/defenses, movement/opportunity, initiative, and action-economy rule skills.
- Do not invent unsupported mechanics; mark any uncertain behavior as suggested ruling.
