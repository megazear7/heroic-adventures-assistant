# Rule: Races and Ancestry Features

## Scope

Use this rule for race selection, racial advancement cadence, racial movement/health identity, and racial feature timing.

## Consolidated Rule

Each race defines:
- baseline stat profile
- movement profile (major/minor)
- creation health
- advancement health trend
- unique racial features with action tags

Race is not cosmetic; it materially shapes initiative, durability, mobility, and feature access.

## Build Impact Areas

- Frontline durability (dwarf/half-orc patterns)
- Mobility and initiative control (halfling/elf patterns)
- utility/knowledge feature access (human/gnome/elf patterns)

## Progression Integration

- Race advancement levels grant race feature or feat options and health adjustments.
- Racial feature load must still obey global feature/action slot limits.

## Adjudication Notes

- Resolve race creation effects before background/stat package choices to keep sequencing consistent.
- When a racial feature has action type, enforce timing and per-card/per-round limits from action economy rules.
