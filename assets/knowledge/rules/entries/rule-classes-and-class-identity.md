# Rule: Classes and Class Identity

## Scope

Use this rule for class feature interpretation, class role identity, and class advancement choices.

## Consolidated Rule

Class determines primary combat style, feature cadence, and action-type profile. Class advancement grants class feature or feat options plus class-specific health scaling.

## Core Class Axes

- Martial control/durability (fighter, paladin, barbarian)
- Tactical precision and utility (rogue, ranger)
- Divine support and leadership (cleric, priest)
- Arcane or life-powered casting (wizard, warlock, druid)

## Feature Interpretation Pattern

When adjudicating class feature text:
1. Identify action type.
2. Identify target/stat contest.
3. Identify usage gates (heroic uses, once-per-round wording, positional requirements).
4. Apply normal combat timing and trigger rules.

## Progression and Build Safety

- Track feature-slot saturation.
- Avoid selecting redundant features with same tactical niche unless intentional.
- Maintain class identity while allowing feat customization.

## Adjudication Notes

- If class and feat options conflict on timing, use explicit feature wording first.
- For hybrid builds, ensure weapon training and spell economy support chosen feature package.
